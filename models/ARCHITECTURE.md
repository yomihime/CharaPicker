# models 架构说明

## 负责什么

- 放置本地推理模式需要的模型文件或模型目录。
- 为后续本地 AI 推理提供统一查找位置。

## 不负责什么

- 不放 Python 源码。
- 不放项目素材、缓存、知识库或导出结果。
- 不放外部二进制工具。
- 不放 UI 文案或界面资源。

## 关键文件

- `.gitkeep`：保留空目录。
- 当前未提交具体模型文件。

## 与其他目录的关系

- `gui/pages/settings_page.py` 当前提供模型路径输入框。
- `utils/env_manager.py` 后续可检测模型路径和可用性。
- `core` 后续本地推理流程可读取用户配置指向的模型。

## 维护注意事项

- 模型文件通常体积较大，默认不提交到 Git。
- 新增示例配置时只写路径说明，不提交真实权重。
- 保持模型目录命名清晰，便于用户在设置页中选择。
- 发布包是否包含模型应由发布策略单独决定。
