# 多素材平级接入前重构解耦计划框架（zh_CN）

> 本文档只是计划框架，不代表正式执行计划，也不代表当前实现状态。
> 后续真正准备执行第二阶段重构时，应重新核对当前代码和第一阶段 P1 的完成结果，再把本文扩写为正式计划。

最近整理日期：2026-06-05。

计划阶段：框架占位。

可用性：仅用于后续讨论和正式计划起草，不作为直接实施依据。

## 1. 开发背景

自上一次结构整理后，项目已经经历正式提取模式、角色卡页面、角色卡质量调优、日志治理和多个 UI/体验修复。代码能力明显前进，但提取主线仍主要围绕视频素材演进。

后续 P0 需要支持文本、字幕、音频转写、图片、漫画和混合媒体。它们不应作为视频流程的附属分支接入，而应在素材抽象、扫描、提取计划、知识库写入和证据引用上成为平级素材类型。

因此第二阶段先做架构体检与解耦框架准备，再进入多素材功能开发。

## 2. 当前状态

- 视频预览和正式视频提取已有基础闭环。
- 音频转写已有正式提取相关能力，但预览链路和统一知识库消费仍未完整打通。
- 文本、字幕、图片、漫画和混合媒体尚未作为平级素材类型进入完整 Extract Once 工作流。
- 当前尚未形成正式重构计划；本文件只记录为什么需要重构、重构前应关注哪些边界。

## 3. 框架范围

第二阶段关注多素材接入前的架构体检和解耦准备，重点观察：

- 素材类型是否被视频流程隐性主导。
- 素材扫描、处理、提取、知识库写入和 GUI worker 之间是否存在过强耦合。
- 视频、音频、文本、图片和漫画作为平级素材时，需要哪些共同边界。
- 哪些旧兼容路径、阶段性 helper 或页面直连逻辑会影响后续扩展。

## 4. 非目标

- 本框架不定义具体重构步骤。
- 本框架不新增数据模型、迁移规则或目录结构。
- 本框架不直接指导代码改动。
- 本框架不设计 P0 多素材功能的具体实现。
