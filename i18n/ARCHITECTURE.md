# i18n 架构说明

## 负责什么

- 放置用户可见文案的多语言 JSON 文件。
- 为界面、提取流程、编译流程、模型测试和设置反馈提供本地化字符串。
- 维护当前支持的基础语系。

## 不负责什么

- 不放业务逻辑。
- 不放 UI 布局代码。
- 不放运行时图片、颜色或主题资源。
- 不保存用户项目数据。

## 关键文件

- `zh_CN.json`：简体中文，默认语系。
- `zh_TW.json`：繁体中文。
- `en_US.json`：英语。
- `ja_JP.json`：日语。

## 与其他目录的关系

- `utils/i18n.py` 负责读取本目录 JSON，并提供 `t()` 给其他模块使用。
- `gui/` 的用户可见文案应通过 `t()` 获取。
- `core/` 的洞察、提取、编译和错误提示文案也通过 `t()` 获取。
- `gui/pages/model_page.py`、`gui/pages/prompt_page.py` 和设置页新增文案时必须同步维护本目录。

## 维护注意事项

- 新增 UI 文案时，四个语系文件都要同步补键。
- key 使用稳定、可读的命名，避免复用语义不同的字符串。
- JSON 文件统一使用 UTF-8 编码。
- 文案可以保持轻微拟人化，但必须优先清晰表达当前状态。
