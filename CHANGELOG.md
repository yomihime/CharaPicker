## v0.8.0-beta - 2026-07-21

### 新增

- 正式提取引入 `FormalExtractionRunPlan` 主索引，统一视频、图片、音频和文本四种媒体类型的素材扫描、预览采样、正式分派、知识库聚合与来源追踪。
- 新增普通文本、SRT/ASS timed text、静态图片、漫画页组、音频 transcript、视频字幕关联和补充型原生音视频理解 handler；不支持的模型能力或格式会返回可解释 warning，不阻断其它素材。
- 角色卡证据新增媒体类型、内容形态、`source_trace`、`extraction_run_id` 和 `source_context.source_runs`，便于从角色卡追溯到正式提取 run 与原始素材。
- 新增结构化提取失败样例、拒绝样例和洞察媒体 metadata，便于定位模型拒绝、解析失败和多内容形态分派问题。
- 新增 ZIP、CBZ、EPUB、文本型 PDF、7z、RAR 与 CBR 的受控输入预处理，派生素材继续复用现有 text/image 提取链路。

### 变更

- transcript 统一建模为从音频或视频派生的 text 成果；原生音视频理解只补充语气、环境声、音乐和画面线索，不替代逐字 transcript。
- 输入容器原件保留在 `raw/`，派生材料按 raw 相对路径写入 `materials/derived_inputs/`，预处理 manifest 继续使用 schema v1 并记录来源、派生项、大小、fingerprint、warning 和失败项。
- 归档预处理增加路径穿越、文件名碰撞、加密内容、解压炸弹、嵌套容器和损坏文件保护；通用 ZIP/7z/RAR 不再展开视频，CBZ/CBR 只接纳漫画图片页。
- 非原始素材处理只在存在显式导入的视频时要求 FFmpeg；缺少 FFmpeg 时可以取消、忽略全部视频继续处理其它素材，或下载 FFmpeg 后自动执行。
- 项目页素材状态改用轻量完整性检查，避免频繁计算 SHA-256 阻塞 UI；复用判定和 raw 清理前的强校验仍保留，并把 raw 清理移入 Qt 工作线程。

### 修复

- 修复模型 provider 声明原生音频能力时，普通文本或视觉模型可能被错误分派到 native audio handler 的问题。
- 修复角色卡编译证据可能丢失来源 run、正式聚合可能混入非当前 run 产物的问题。
- 修复不同 raw 子目录下同名容器可能共享派生目录或 manifest 的问题，并补充派生文件大小与 SHA-256 fingerprint 校验。
- 修复归档内视频可能隐式进入 FFmpeg 流程、缺少 FFmpeg 时提前阻断非视频预处理，以及 FFmpeg 不可用时无法继续处理其它素材的问题。

### 文档

- 完成并归档提取质量与可观测性、正式提取架构重构、多内容形态覆盖和更多输入格式支持计划；更新 TODO、架构说明、提取工作流、README 与第三方许可说明。
- 新增统一多内容形态离线回归、各媒体 handler、输入预处理、GUI 状态、i18n 和真实素材验收脚本。

### 构建

- 新增 `pypdf>=6.14.2,<7` 运行时依赖，用于文本型 PDF 提取；7z/RAR/CBR 复用用户安装或项目本地的可选 7-Zip，不由应用自动下载或随默认源码依赖安装。
- 默认应用版本升级为 `v0.8.0-beta`，同步更新项目元数据、构建默认值、README、多语言 README、关于页版本文案和发布规范示例。

## v0.7.0-beta - 2026-06-05

### 新增

- 角色卡正式编译新增分层证据包：`direct_evidence_episodes`、`mention_evidence_episodes`、`causal_context_episodes` 和 `season_context`，并把结果写入 CharaPicker 扩展字段，便于追踪角色事实、提及、因果背景和季级上下文。
- 角色卡编译新增 AI 别名校验与重分类链路：当用户填写中文名或别名而知识库使用英文/罗马字候选名时，可从 `episode_content.targets` 解析已验证别名并重新匹配 direct 证据。
- 角色卡质量诊断新增 `needs_review_reasons`、`conflict_groups`、`alias_resolution` 和 `parse_diagnostics`，用于解释证据不足、低置信别名、知识库 warnings、冲突复核和 AI JSON 修复情况。

### 变更

- 没有 direct 证据的正式角色卡仍会编译失败，避免为未出场或无证据角色硬生成资料卡；`targets` 只作为别名候选和辅助信息，不单独算 direct。
- 角色卡 AI 复核 prompt 新增 `evidence_layers` 变量，beta 阶段旧的用户 prompt override 可能需要重置后才能使用新分层证据。
- `quality.warnings` 只保留用户可读 warning，结构化复核原因保存在 `extensions["charapicker"]["quality_checks"]["needs_review_reasons"]` 并由 UI 本地化展示。
- 调整模型调用、素材处理、正式提取和角色卡编译日志：INFO 记录阶段起止与结果摘要，DEBUG 记录计数、分支和 token/响应形状，WARNING/ERROR 区分可恢复降级与任务失败。
- 收紧日志中的敏感输出：模型端点统一脱敏，FFmpeg 不再打印完整命令，素材导入与知识库读取失败日志避免记录完整本地路径。

### 修复

- 修复右侧洞察流默认提示或较长事件文本可能撑出面板宽度、出现横向溢出的问题。
- 修复成功重编译后 stale reason 或失败状态残留的问题，已编译卡片重新成功后会清理旧错误与 stale warning。
- 修复结构化复核原因可能以内部 reason key 形式混入普通 warning 或导出展示的问题。

### 文档

- 更新四语种提取工作流参考文档和角色卡编译 Mermaid 流程图，补充分层证据、别名重分类、质量诊断和未出场角色失败保护。
- 更新根目录、core、gui、projects 和 docs 架构说明，明确角色卡证据分层、质量诊断扩展字段和 GUI/core 边界。
- 归档角色卡质量后续执行计划，并更新当前 TODO 与计划索引，将已完成的角色卡质量调优和日志治理移出队列，剩余工作收敛为素材覆盖、提取验证和可选增强。
- 更新运行时中间件、utils 架构说明和当前 TODO，沉淀日志等级划分、脱敏边界与关键链路观测点。
- 新增提取质量与可观测性、多素材前重构解耦和多素材种类支持三阶段计划框架，确认“P1 稳定性 -> 多素材前解耦 -> P0 多素材支持”的后续路线。

### 构建

- 默认应用版本升级为 `v0.7.0-beta`，同步更新 `pyproject.toml`、`utils/app_metadata.py`、`build.bat`、README、多语言 README、关于页版本文案和发布规范示例。

## v0.6.0-beta - 2026-06-04

### 新增

- 增加正式素材提取的三种入口模式：完整提取、洁净提取和快速提取。完整/洁净模式走高质量线性流程；快速模式支持用户确认并发数后并行提取 chunk，再用 AI 重整理集和季。
- 增加正式提取的 run 隔离与产物元数据，chunk、episode、season 产物记录 `extraction_run_id`、`extraction_stage`、`schema_version`、上下文策略和 token 用量，聚合时只消费当前 run 中合格的正式产物。
- 增加线性上下文构建能力：正式 chunk 提取会带同集已完成 chunk、同季历史 episode 和跨季长摘要，并按模型上下文窗口、128k 历史池、输出预留和安全余量控制预算。
- 增加 AI episode/season 整理链路：每集完成后由 AI 合并 `episode_content` 并生成 `episode_summary`，每季完成后由 AI 合并 `season_content` 并生成 `season_summary`。
- 增加可再生提取产物清理能力，洁净提取会清理中间文件后重新跑正式流程，并在新正式知识库写入成功后把已编译正式角色卡标记为 stale。
- 增加正式提取回归验证脚本 `scripts/validate_formal_extraction_workflow.py`，覆盖 run 隔离、模式分流、上下文预算、清理边界和角色卡 stale 标记等基础行为。

### 变更

- 正式提取进度事件现在展示更细的 run、season、episode、chunk 和 AI 整理阶段信息，项目页洞察流能区分完整、洁净和快速流程。
- 模型 preset 增加 `context_window_tokens`，用于估算上下文预算；未配置时使用保守默认值并记录策略标记。
- 角色卡编译继续只读取正式知识库；直接匹配失败时可从 `episode_content.targets` 候选中解析别名，但 AI 返回的别名必须实际存在于知识库候选中。
- 角色卡编译成功后会清理旧的错误状态，避免成功重编译后页面仍残留失败提示。

### 修复

- 修复角色卡使用中文名、别名或罗马音而知识库使用英文/候选名时可能无法匹配的问题。
- 修复角色卡编译成功后 `last_error`、warning 或失败态未及时清空导致的误导性状态展示。
- 修复正式提取失败重跑时旧产物可能混入聚合结果的风险。

### 文档

- 更新四语种提取流程参考文档，补充完整/洁净/快速提取流程图、角色卡编译流程图、跨 chunk/跨集/跨季上下文策略和当前限制。
- 归档正式素材详细提取流程与模式计划，将剩余角色卡上下文分层、验证脚本和日志治理残项迁入当前 TODO。
- 新增角色卡质量后续临时计划，覆盖 TODO P1 的角色卡编译上下文分层、冲突消解和质量评估强化。
- 更新 README、多语言 README、发布规范和产品说明中的当前版本到 `v0.6.0-beta`。

### 构建

- 默认应用版本升级为 `v0.6.0-beta`，同步更新 `pyproject.toml`、`utils/app_metadata.py` 和 `build.bat`。
