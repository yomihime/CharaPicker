# bin 架构说明

## 负责什么

- 放置运行时需要的外部二进制工具。
- 后续可存放 ffmpeg、llama 等本地可执行文件或其适配目录。
- 为离线或本地模式提供工具发现位置。

## 不负责什么

- 不放 Python 源码。
- 不放用户素材、项目数据或导出结果。
- 不放模型权重文件。
- 不放 UI 文案或资源说明。

## 关键文件

- `.gitkeep`：保留空目录。
- 当前未提交具体二进制文件。

## 与其他目录的关系

- `utils/env_manager.py` 后续应检测本目录下工具是否存在和版本是否可用。
- `core` 后续媒体解析流程可通过环境检测结果调用这些工具。
- 打包流程应决定哪些工具需要进入发布包。

## 维护注意事项

- 大型二进制文件默认不提交到 Git。
- 新增工具时记录来源、版本和许可信息。
- 不要把临时下载文件留在本目录。
- 发布包只包含运行确实需要的工具。
